# 有限会社頼鉄工所 研削加工計算ポータル

有限会社頼鉄工所の研削加工支援ポータルです。社内従業員がPC、タブレット、スマホから加工条件をすばやく確認できるように、HTML、CSS、JavaScriptのみで構成しています。

このサイトはGitHub Pagesで公開する前提の静的Webサイトです。外部公開されても問題ないように、個人名、従業員情報、取引先名、図面番号、加工単価、社内機密情報は入れません。

## 現在実装済み機能

- 円筒研削条件計算
  - 砥石周速 a（m/min）
  - 砥石直径 b（mm）
  - 切込み e（mm）
  - 主軸回転数 f（rpm）
  - 荒切込（mm）
  - 精研切込（mm）
  - 仕上切込（mm）
- 重量計算
  - 鉄系材料専用
  - 丸棒
  - 中空丸棒（パイプ）
  - 固定密度 7.85 g/cm³
  - 重量（kg）
- テーパー角度計算
  - 角度を求める
  - 径を求める
  - 長さを求める
  - 片側角度と全角度を表示
- JISはめあい公差検索
  - 呼び径（mm）
  - 公差記号
  - 上限公差、下限公差（μm）
  - 上限寸法、下限寸法、狙い値（mm）

## 今後追加予定

- 平面研削条件計算

## GitHub Pages公開方法

このリポジトリでは `gh-pages` ブランチをGitHub Pages公開用ブランチとして使います。

1. GitHubで `grinding-calculation-portal` リポジトリを開きます。
2. `Settings` を開きます。
3. `Pages` を開きます。
4. `Build and deployment` の `Source` を `Deploy from a branch` にします。
5. `Branch` を `gh-pages`、フォルダを `/root` にします。
6. `Save` を押します。

公開URL:

```text
https://yori-3.github.io/grinding-calculation-portal/
```

## ローカル実行方法

特別なビルドは不要です。

1. このリポジトリをPCへ取得します。
2. `index.html` をブラウザで開きます。

ローカルで確認するだけなら、サーバー起動や追加ライブラリは不要です。

## 円筒研削条件計算の計算式

```text
主軸回転数 f（rpm） = (60 * a / Math.PI / b) * 1000
荒切込 = f * e
精研切込 = 荒切込 / 3
仕上切込 = 荒切込 / 9
```

表示桁数:

- 主軸回転数 f: 小数1桁
- 荒切込: 小数3桁
- 精研切込: 小数3桁
- 仕上切込: 小数3桁

## 重量計算の計算式

鉄系材料専用として、密度は `7.85 g/cm³` 固定です。

丸棒:

```text
体積（mm³） = π × (D / 2)^2 × L
重量（kg） = 体積 × 7.85 ÷ 1,000,000
```

中空丸棒（パイプ）:

```text
体積（mm³） = π × ((D / 2)^2 − (d / 2)^2) × L
重量（kg） = 体積 × 7.85 ÷ 1,000,000
```

表示桁数:

- 重量: 小数3桁

注意:

本計算は鉄系材料（密度7.85g/cm³）の概算重量です。実際の重量は材質、寸法公差、穴加工などにより変動します。

## テーパー角度計算の計算式

角度は片側角度を基準に計算します。全角度は片側角度の2倍です。

### 角度を求める

```text
径差 = D - d
片側角度 θ（度） = atan((D - d) / 2 / L) を度へ変換
全角度 = θ × 2
勾配 = (D - d) / L
```

表示項目:

- 径差（mm）
- 片側角度（°）
- 片側角度（度分秒）
- 全角度（°）
- 全角度（度分秒）
- 勾配（1/○○形式）

### 径を求める

```text
径差 = 2 × L × tan(θ)
小径 = 基準径 − 径差
大径 = 基準径 ＋ 径差
全角度 = θ × 2
```

選択した方向により、大径から小径、または小径から大径を求めます。

### 長さを求める

```text
径差 = D - d
テーパー長 L = (D - d) / 2 / tan(θ)
全角度 = θ × 2
```

表示桁数:

- 角度: 小数4桁
- 径差、径、長さ: 小数3桁

注意:

本計算は一般的な円錐テーパーを対象としています。角度は片側角度を基準に計算しています。図面が片側角度か全角度かを確認して使用してください。

## JISはめあい公差検索

JISはめあい公差検索は、添付PDF `1.pdf` を元データとして作成した `tolerance-data.js` を参照します。

Webアプリ実行時にはPDFやExcelを読み込みません。GitHub Pages上では、組み込み済みのJavaScriptデータだけで検索します。

### 登録データ

- 軸公差: `toleranceData.shaft`
- 穴公差: `toleranceData.hole`
- 公差値の単位: μm
- 径範囲判定: `最小径超 < 呼び径 <= 最大径以下`

PDF上で `-` または空欄になっている径範囲は、推測せず未登録です。

### 計算式

```text
上限公差mm = 上限公差μm ÷ 1000
下限公差mm = 下限公差μm ÷ 1000

上限寸法 = 呼び径 + 上限公差mm
下限寸法 = 呼び径 + 下限公差mm

狙い公差mm = (上限公差mm + 下限公差mm) ÷ 2
狙い値 = 呼び径 + 狙い公差mm
```

表示桁数:

- 公差: μm表記
- 上限寸法、下限寸法、狙い値: 小数3桁

### 公差データ更新手順

1. 元データPDFを確認します。
2. 読み取れた公差値だけを `tolerance-data.js` に登録します。
3. 推測、補完、外部サイトの値は使用しません。
4. 読み取れない箇所は登録せず、`tolerance-check.md` の要確認一覧に記載します。
5. `tolerance-check.md` を更新し、登録範囲と公差値を確認します。
6. `script.js` と `tolerance-data.js` の構文チェックを行います。
7. 既存機能とJISはめあい公差検索の動作確認を行います。
