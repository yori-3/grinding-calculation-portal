# YORI加工計算

有限会社頼鉄工所の加工計算サイトです。社内従業員がPC、タブレット、スマホから加工条件をすばやく確認できるように、HTML、CSS、JavaScriptのみで構成しています。

このサイトはGitHub Pagesで公開する前提の静的Webサイトです。外部公開されても問題ないように、個人名、従業員情報、取引先名、図面番号、加工単価、社内機密情報は入れません。

## 現在実装済み機能

- 円筒研削条件計算
  - 品物周速 a（m/min）
  - 品物直径 b（mm）
  - 切込み e（mm/rev）
  - 主軸回転数 f（rpm）
  - 荒入力値（mm/min）
  - 精研入力値（mm/min）
  - 仕上入力値（mm/min）
- 重量計算
  - 鉄系材料専用
  - 丸棒
  - 中空丸棒（パイプ）
  - 固定密度 7.85 g/cm³
  - 重量（kg）
- テーパー角度計算
  - 角度を求める
  - 径を求める
  - 長さを求める
  - 軸方向長さ、直線距離、勾配入力に対応
  - 片側角度と全角度を表示
- はめあい公差
  - 呼び径
  - 公差記号
  - 上限公差、下限公差（mm換算）
  - 上限寸法、下限寸法、狙い値

## 今後追加予定

- 平面研削条件計算

## GitHub Pages公開方法

このリポジトリでは `gh-pages` ブランチをGitHub Pages公開用ブランチとして使います。

このサイトはVite、React、npm buildを使わない静的HTMLサイトです。GitHub Pagesの公開方式は、A案の「`gh-pages` ブランチから公開する方式」に統一します。

リポジトリに `.github/workflows` は置きません。GitHub Pagesが自動で表示する `pages-build-deployment` は、ブランチ公開方式でもGitHub側が内部で実行する公開ジョブです。

`.nojekyll` を配置し、Jekyll変換を行わず静的ファイルをそのまま公開します。

1. GitHubで `grinding-calculation-portal` リポジトリを開きます。
2. `Settings` を開きます。
3. `Pages` を開きます。
4. `Build and deployment` の `Source` を `Deploy from a branch` にします。
5. `Branch` を `gh-pages`、フォルダを `/root` にします。
6. `Save` を押します。

`Source` を `GitHub Actions` に変更しないでください。Actions方式にする場合は、別途 `.github/workflows` とデプロイ設定を追加する必要があります。

公開URL:

```text
https://yori-3.github.io/grinding-calculation-portal/
```

## 円筒研削条件計算の計算式

```text
主軸回転数 f（rpm） = (60 * a / Math.PI / b) * 1000
荒入力値 = f * e
精研入力値 = (f * e) / 3
仕上入力値 = (f * e) / 9
```

表示桁数:

- 主軸回転数 f: 小数1桁
- 荒入力値: 小数3桁
- 精研入力値: 小数3桁
- 仕上入力値: 小数3桁

## 重量計算の計算式

鉄系材料専用として、密度は `7.85 g/cm³` 固定です。

丸棒:

```text
体積（mm³） = π × (D / 2)^2 × L
重量（kg） = 体積 × 7.85 ÷ 1,000,000
```

中空丸棒（パイプ）:

```text
体積（mm³） = π × ((D / 2)^2 − (d / 2)^2) × L
重量（kg） = 体積 × 7.85 ÷ 1,000,000
```

表示桁数:

- 重量: 小数3桁

注意:

本計算は鉄系材料（密度7.85g/cm³）の概算重量です。実際の重量は材質、寸法公差、穴加工などにより変動します。

## テーパー角度計算の計算式

角度は片側角度を基準に計算します。全角度は片側角度の2倍です。

### 角度を求める

```text
径差 = D - d
半径差 r = (D - d) / 2

軸方向長さ L の場合:
片側角度 θ = atan(r / L)

直線距離 S の場合:
片側角度 θ = asin(r / S)

勾配 1:X の場合:
片側角度 θ = atan(1 / (2X))

全角度 = θ × 2
```

表示項目:

- 径差（mm）
- 半径差（mm）
- 片側角度（°）
- 片側角度（度分秒）
- 全角度（°）
- 全角度（度分秒）
- 軸方向長さ（mm）
- 直線距離（mm）
- 勾配入力時は勾配（1:X）

### 径を求める

```text
軸方向長さ L の場合:
径差 = 2 × L × tan(θ)

直線距離 S の場合:
径差 = 2 × S × sin(θ)

小径 = 基準径 − 径差
大径 = 基準径 ＋ 径差
全角度 = θ × 2
```

角度入力は片側角度または勾配 1:X から選択できます。選択した方向により、大径から小径、または小径から大径を求めます。

### 長さを求める

```text
径差 = D - d
半径差 r = (D - d) / 2
軸方向長さ L = r / tan(θ)
直線距離 S = r / sin(θ)
全角度 = θ × 2
```

角度入力は片側角度または勾配 1:X から選択できます。

表示桁数:

- 角度: 小数4桁、度分秒も表示
- 径差、半径差、径、軸方向長さ、直線距離: 小数3桁

注意:

軸方向長さは中心線方向の長さです。直線距離はテーパー面に沿った長さです。勾配は1:Xで入力してください。図面寸法を確認して使用してください。

## はめあい公差

はめあい公差は、添付PDF `1.pdf` と `軸の基礎となる寸法許容差の数値表.xlsx` を元データとして作成した `tolerance-data.js` を参照します。

Webアプリ実行時にはPDFやExcelを読み込みません。GitHub Pages上では、組み込み済みのJavaScriptデータだけで検索します。

### 登録データ

- 軸公差: `toleranceData.shaft`
- 穴公差: `toleranceData.hole`
- 登録公差値の単位: μm
- 画面表示する上限公差、下限公差: mm換算
- 径範囲判定: `最小径超 < 呼び径 <= 最大径以下`
- 軸公差 `x7`、`x8` を追加

PDFまたはExcel上で `-` または空欄になっている径範囲は、推測せず未登録です。

### 計算式

```text
上限公差mm = 上限公差μm ÷ 1000
下限公差mm = 下限公差μm ÷ 1000

上限寸法 = 呼び径 + 上限公差mm
下限寸法 = 呼び径 + 下限公差mm

中央値mm = (上限公差mm + 下限公差mm) ÷ 2

軸公差の狙い公差mm = 中央値mm + 0.001
穴公差の狙い公差mm = 中央値mm - 0.001

狙い値 = 呼び径 + 狙い公差mm
```

表示桁数:

- 上限公差、下限公差: mm換算で小数3桁。ただし0は `0` 表示
- 上限寸法、下限寸法、狙い値: 小数3桁

### 公差データ更新手順

1. 元データPDFを確認します。
2. 読み取れた公差値だけを `tolerance-data.js` に登録します。
3. 推測、補完、外部サイトの値は使用しません。
4. 読み取れない箇所は登録せず、`tolerance-check.csv` の要確認行に記載します。
5. `tolerance-check.csv` を更新し、登録範囲と公差値を確認します。
6. `script.js` と `tolerance-data.js` の構文チェックを行います。
7. 既存機能とはめあい公差の動作確認を行います。
